**Лабораторная работа №4**

Изучение основ проектирования распределенных систем. Получение
практических навыков декомпозиции монолитного приложения на
микросервисы, описания контрактов взаимодействия (IDL) и организации
межсервисной коммуникации с использованием технологии gRPC.

Разделена существующая информационная система на два независимых
микросервиса, которые взаимодействуют друг с другом исключительно по
сети через gRPC.

Реализована следующая архитектура:
1. Service A (Core Service):
Содержит основную бизнес-логику турагентсва (travelagency4)
2. Service B (Reference Service):
Содержит справочную информацию, каталоги, валидационные данные. (travelagency-service-b)

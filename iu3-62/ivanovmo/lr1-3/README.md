# Brainfuck Superior Experience

Этот проект расширяет классический интерпретатор Brainfuck, добавляя поддержку объектно-ориентированного программирования **без введения новых синтаксических команд**. Все ООП-возможности реализованы через специальный протокол работы с памятью. Любая стандартная программа на Brainfuck продолжает работать как обычно при соблюдении определенных условий.

## Как это работает

В интерпретаторе зарезервированы первые несколько ячеек памяти для управляющих регистров:

| Адрес | Назначение |
|-------|------------|
| `cell[0]` | Код операции ООП (0 – нет действия, 1 – создать объект, 2 – вызвать метод) |
| `cell[1]` | Идентификатор класса / объекта |
| `cell[2]` | Идентификатор метода (при вызове) |
| `cell[3]` | Указатель на параметры / количество параметров |

После выполнения **каждой** обычной команды Brainfuck интерпретатор проверяет `cell[0]`. Если там не 0, выполняется соответствующее ООП-действие, после чего `cell[0]` обнуляется. Таким образом, программа может «вызвать» ООП-операцию, пр# Brainfuck + ООП (без изменения синтаксиса)

## Как это работает

В интерпретаторе зарезервированы первые несколько ячеек памяти для управляющих регистров:

| Адрес | Назначение |
|-------|------------|
| `cell[0]` | Код операции ООП (0 – нет действия, 1 – создать класс, 2 – создать объект, 3 – вызвать метод, 4 – добавить метод) |
| `cell[1]` | Идентификатор класса / объекта |
| `cell[2]` | Идентификатор метода (при вызове) |
| `cell[3]` | Указатель на параметры / количество параметров |

## Установка

Скопируйте файл интерпретатора `brainfuck.py` (см. раздел «Реализация»). Убедитесь, что у вас установлен Python 3.6+.

```bash
python brainfuck.py program.bf
```
